# Engram v5.0.0 — Cleaned Baseline

> **Published:** 2026-04-23 · **Tag:** `v5.0.0`

Engram v5.0.0 is a deliberate reverse-migration from the broken dynamic "learning memory" stack to a static-only, explicit-write system. The release removes the non-functional derived subsystems, keeps only the durable entities that actually worked in production, and introduces a simpler session-start path with version negotiation and offline cache fallback.

## Highlights

### Static-only storage model

v5 keeps six explicit entity classes:
- issues
- documents
- memories
- behavioral rules
- credentials / vault
- API tokens

Everything else that depended on automatic extraction, inferred ranking, graph traversal, reranking, or continuous maintenance has been removed from the core product path.

### Observations split completed

The old observations-era runtime has been retired. Data and runtime responsibilities are now split into dedicated static entities:
- `memories`
- `behavioral_rules`
- `credentials`

This removes the previous god-table semantics where one storage model attempted to act as facts, rules, extraction output, scoring substrate, and consolidation artifact simultaneously.

### Session-start simplified

US13 lands in this release:
- gRPC `GetSessionStartContext` returns a static composite payload for session start
- gRPC `NegotiateVersion` enables explicit major-version compatibility checks
- plugin session-start now uses the static path with local cache fallback
- cache is stored at `${ENGRAM_DATA_DIR}/cache/session-start-{project-slug}.json`
- when the server is unreachable, Claude Code receives cached context with a visible staleness banner instead of silent failure

### What was removed

The v5 cleanup drops the previously broken dynamic subsystems, including:
- patterns subsystem
- graph subsystem
- learning / scoring / maintenance / consolidation loops
- reranking / HyDE / search-manager stack
- embeddings storage and vector-era runtime dependencies
- raw log / prompt / summary / indexed-session persistence paths that no longer fit the static model

## Breaking Changes

### Architecture

- Engram is now a static-first memory system.
- The old dynamic relevance / learning / graph model is no longer part of the production path.
- Session-start inject is simplified to static entities only: issues, behavioral rules, memories.

### Data model

- observations-era runtime behavior is removed from the main product flow
- dedicated static stores are the source of truth for memory/rule/credential reads
- session-start cache is now a first-class client behavior

### Version compatibility

- v5 introduces explicit major-version negotiation for the session-start path
- mixed major client/server pairs must fail clearly instead of degrading silently

### Plugin/runtime versions

- plugin version: `5.0.0`
- daemon version: `v5.0.0`
- no split-version deployment is supported for the release ceremony

## Upgrade Notes

1. Upgrade the Engram plugin to `5.0.0`.
2. Upgrade the daemon binary to `v5.0.0`.
3. Ensure the updated plugin package is synced to the marketplace.
4. Restart Claude Code and the daemon.
5. Verify session-start inject, cache fallback, and plugin update detection on a clean install.
