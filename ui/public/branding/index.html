8. Choosing solid vs transparent

Context Use
App icon, browser tab, marketing card, OG image engram-icon*.svg (solid)
Header on already-dark UI, sidebar logo *-transparent.svg
Mention in body text, README badge favicon-transparent.svg at 16-24 px
Photo / video / patterned overlay Solid (the canvas guarantees contrast)

9. Don'ts

  • Don't recolor the orange mark — use #EE7410 only.
  • Don't apply drop shadows, gradients, or strokes to the SVG.
  • Don't stretch — the icon is square; pad container instead.
  • Don't use the 1024 master inline at small sizes — pick the closest optimized variant.
  • Don't use favicon.svg below 48 px — use favicon-32.svg or favicon-16.svg for crisp pixel rendering.
  • Don't drop the canvas on photos or busy backgrounds — use solid for guaranteed contrast.
  • Don't put the transparent mark on a light or orange surface — picks up no contrast.
